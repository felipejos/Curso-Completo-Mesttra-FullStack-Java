# Orientações

---

## Contexto

Atualmente, existe muito conteúdo gratuito ou de baixo custo (udemy e outras plataformas) disponível na Internet, sobre diversos assuntos na área de tecnologia e/oudesenvolvimento.

Uma das grandes dificuldades que os iniciantes e até intermediários encontram, é saber qual trilha de estudo que o mesmo deve seguir.

---

## Referência principal

Neste site (https://roadmap.sh/), existem diversos roteiros de estudos e sugerimos que você utilize estes guias como referência para a ordem do que estudar. Cada caixa da trilha de conhecimento se for clicada, exibirá os tópicos mais relavantes do que deve ser estudado.

---

## Como usar a trilha

Use os termos/conceitos/tópicos apresentados para buscar vídeos, artigos, cursos sobre o tema, para que você possa se aprofundar.

---
# Complemento da Lição

## 1) Como transformar uma trilha em plano de estudos (passo a passo)
1) Escolha **uma** trilha do roadmap (ex.: Backend, Java, Full Stack).
2) Leia o roadmap de cima para baixo e marque:
   - **Essencial** (base)
   - **Importante** (para trabalhar)
   - **Avançado** (para depois)
3) Para cada caixa (tópico), faça 3 coisas:
   - defina o **objetivo** (o que você precisa conseguir fazer)
   - pegue **1 material principal** (curso ou playlist)
   - pegue **1 material de apoio** (artigo/documentação)
4) Feche cada tópico com:
   - **mini-projeto** ou **exercício prático**
   - anotação de dúvidas (para revisar)

---

## 2) Regra simples para não se perder (prioridade)
- Primeiro: **fundamentos** (lógica, linguagem, Git, terminal)
- Depois: **HTTP + APIs + banco + testes**
- Depois: **arquitetura (MVC/Clean) + boas práticas**
- Por último: **cloud, deploy, mensageria, escalabilidade**

---

## 3) Aplicando no seu contexto (contatos + MySQL + API)
Um caminho natural (em ordem) para seu projeto evoluir:

1) Console CRUD (ok)
2) Console + MySQL (ok)
3) MVC no console (em andamento)
4) Transformar em API HTTP (próximo)
5) Consumir via frontend (com testes via Thunderclient)
6) Organizar com camadas (Controller/Service/Repository) e testes

---

## 4) Exercício rápido (fixação)
Escolha **uma trilha** do roadmap.sh e faça:
- Liste 5 caixas (tópicos) em ordem
- Para cada tópico, escreva:
  - “O que eu preciso conseguir fazer?” (1 frase)
