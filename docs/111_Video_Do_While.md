## Complemento: como assistir e “tirar o máximo” desse vídeo (passo a passo)

### 1) O que você precisa identificar no exemplo do vídeo
Enquanto assiste, tente pausar e localizar sempre estas 3 partes:

- **Bloco do `do { ... }`**  
  É o que “acontece” a cada repetição (imprime, pede algo, calcula etc.)

- **Condição do `while (...)`**  
  É o que decide se repete (`true`) ou se para (`false`)

- **Variável de controle**  
  É o valor que muda e influencia a condição (ex: `opcao`, `tentativa`, `numero`, `senha`)

---

### 2) Checklist rápido de entendimento (use como “prova”)
Se você conseguir responder estas perguntas, você entendeu o do...while:

- **O que o programa executa antes de testar a condição?**  
- **Qual é a condição que mantém o loop rodando?**  
- **Qual é o evento que faz o loop parar?**  
- **Onde a variável de controle é atualizada?**

---

### 3) O erro mais comum (pra você ficar atento no vídeo)
- **Esquecer o `;` no final** do `while`:

    do {
        // ...
    } while (condicao);  // tem ; aqui

---

### 4) Exercício rápido (depois do vídeo)
Reescreva o exemplo do vídeo mudando apenas 1 coisa:
- se o vídeo usa `opcao != 0`, troque para `opcao != 9`
- ou troque o menu para “continuar? (s/n)”
- ou troque para “tentar senha até acertar”

A ideia é você treinar o mesmo padrão com outro contexto.

---

## Complemento: frase mental para nunca esquecer
“**Executa uma vez. Depois repete enquanto a condição for verdadeira.**”
