## Complemento da Lição

### 🎯 Para que serve esse roadmap
Usar como trilha para aprender **Front-End** (o que o usuário vê e interage) e conseguir:
- criar páginas com **HTML + CSS**
- adicionar lógica e interação com **JavaScript**
- consumir APIs (ex.: suas APIs em Spring Boot)
- construir projetos com um framework moderno (ex.: **React**)

---

### 🧭 Ordem prática de estudo (sequência recomendada)
1) **Fundamentos da Web**
   - como funciona: navegador, HTTP, URLs
   - DevTools (Inspecionar / Network / Console)

2) **HTML (estrutura)**
   - tags básicas, formulários, semântica (`header`, `main`, `section`, `footer`)
   - acessibilidade básica (labels, alt)

3) **CSS (visual)**
   - box model (margin/padding/border)
   - flexbox e grid
   - responsivo (media queries)

4) **JavaScript (interação)**
   - variáveis, funções, arrays/objetos
   - DOM (selecionar elementos e reagir a eventos)
   - `fetch` + JSON (consumir API)

5) **Git/GitHub**
   - commits, branches, pull request

6) **Framework (React como padrão do mercado)**
   - componentes, props, state
   - roteamento
   - consumo de API + formulários

---

### ✅ Checklist do que você precisa dominar para “trabalhar com API”
- fazer `GET` e mostrar dados na tela
- fazer `POST` com formulário
- tratar erros (mensagem ao usuário)
- loading (carregando) e estados vazios
- validação simples de campos

---

### 🧩 Mini-projetos (para fixar, em ordem)
1) **Página de perfil** (HTML/CSS puro)
2) **Lista de tarefas** (JS + DOM + localStorage)
3) **Catálogo de produtos** (JS + fetch de uma API pública)
4) **Tela de login fake** (front validando campos e simulando chamada)
5) **Dashboard simples** (React consumindo API)

---

### 🔗 Conexão com seu caminho Full-Stack Java
- Back-End (Spring Boot) → cria endpoints REST
- Front-End → consome esses endpoints e exibe no navegador
- Resultado → você vira **Full-Stack** (faz ponta a ponta)
